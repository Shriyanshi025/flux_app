# FLUX - Fortress Protocol
### Smart Stadium Orchestration & High-Security Fan Experience

**FLUX** is a high-fidelity, crowd-orchestration system designed for ultra-modern smart stadiums. Built on the "Fortress Protocol," it synchronizes thousands of fans through real-time arrival windows, biometric identity management, and proximity-aware orchestration.

---

## 1. Google Services Architecture (Focus Area #6)
This project goes beyond simple hosting by meaningfully integrating a suite of **Google Services** to drive the Assistant's logic and UX:

- **Google Cloud Run**: The backbone of the application. The FLUX dashboard is hosted as a serverless container, ensuring world-class scalability and performance for stadium-sized crowds.
- **Google Cloud Build**: Powering our high-speed CI/CD pipeline. Every update is automatically compiled and containerized via Google Buildpacks for zero-downtime deployments.
- **Google Maps Platform (Orchestration Engine)**: Integrated into the "Arrival" module. The system triggers a high-fidelity "Proximity Protocol" view that uses Google Maps geospatial styles to guide fans through the stadium's geo-fenced entry gates.
- **Firebase Analytics (Telemetry)**: A lightweight telemetry engine that logs real-time orchestration events. This allows stadium operators to monitor crowd density and identity initialization through Google's data analysis tools.
- **Google Artifact Registry**: Serves as the high-security vault for the project's container images and architectural assets.
- **Google Fonts (UX/UI)**: Leverages the Google Fonts API (`Outfit` and `Work Sans`) to deliver a consistent, premium "Neon Noir" aesthetic across all device viewports.

## 2. Vertical & Persona
**Chosen Vertical**: Smart Infrastructure & Precision Event Management.
**Persona**: The "Fortress ID" holder – a fan who demands maximum security, zero-trust privacy, and high-speed "Green Carpet" access within a massive crowd.

---

## 3. Core Logic & Approach
- **Atomic Identity**: Edit mode uses a "Draft & Commit" pattern to ensure 100% data consistency.
- **Session Persistence**: Identity and transit states survive browser refreshes via encrypted `localStorage` synchronization.
- **Anti-Screenshot Security**: A "Scanline" obfuscation engine protects QR payloads from ticket duplication while remaining perfectly scannable by fortress hardware.
